Changelog Version 1.10.0.0

Added

  • Profinet View & DCP (requires Npcap or Winpcap):

    • Scans and displays all DCP-enabled devices on the network
    • Identifies specific information about connected Profinet devices
    • Performs specific device actions:
      • Customizes IP Address, Profinet Device Name, Subnet Mask, Gateway
      • Different Reset Modes
      • Wink for visual identification

  • Diagnose View:

    • Scans the Turck devices on the network and flags devices with existing diagnostics
    • Displays the device status or diagnostic message
    • Direct link to the creation of the diagnostic report
    • Accessing the detailed diagnostics page of the affected device

  • Monitoring and visualization of EZ-Array light curtain with light beams that can be switched on and off.

  • Optional: start of a DHCP server with the following functions

    • DHCP Scan List
    • DHCP Start/Stop
    • DHCP Get Adapter List

  • Visualizes the time sequence of process input data of connected analog devices/sensors

  • CODESYS recipes and/or log files (USB data) are uploaded to the PC

Changed

  • Automatic checking and display of new TAS versions
  • The execution of TAS to set the firewall rules without administrator rights can be stopped without the TAS being executed
  • Bar graphs for the display of the RFID tags in the read/write area and the detuning of the RF read/write head in the HF RFID tag actions
  • IP 0.0.0.0 is treated as a default IP address in the network view
  • For compatible IO-Link devices, the product variant ID or description ID is also displayed in the IO-Link Web Part when changing the DeviceID
  • View IP address and Profinet device name when a device is opened via TAS
  • An ARGEE project in the device can be overwritten via the ARGEE view without prior deletion if no protection is set
  • Backup name of the CODESYS application is made up of the name of the CODESYS application, IP address and date
  • Support for Q180/Q300-LNX UHF readers
  • Customization of different icons and texts
  • Adjustments to the website layout

Fixed

  • Malfunction due to CODESYS user login on certain firmware versions
  • Compatibility problems with existing backups of the CODESYS application
  • If more IODDs are linked than the memory size of the web browser allows, the oldest IODD is replaced
  • Fixed Modbus TCP and EtherNet/IP™ memory mapping in web client
  • Added missing channel numbers in Modbus TCP and EtherNet/IP™ memory mapping
  • Display of BEEP support of the FEN20-4DIP-4DOP
  • Incorrect sorting of IP addresses in IO-Link and CODESYS views
  • Excom Gen: Failsafe mode is not displayed on all modules
  • Updated some text messages and tooltips

Changelog Version 1.9.0.0

Added

  • CODESYS View:
  • Scans and displays all CODESYS devices on the network

Changed

  • The user is informed if the started TAS version is still blocked by an older instance
  • The link to the downloads on the product website is country-specific
  • Further differentiation in the address mode between rotary switches and dip switches
  • Adaptation of the design of the web pages

Fixed

  • Updated some text messages and tooltips
  • Repeated capturing of the same HF tag is not indicated
  • Incorrect block transfer with inconsistent parameter sets
  • Incorrect redirection to the device web page after IP address changes or a reset

Changelog version 1.7.0.0

Added

  • IO-Link View:
    • Scans and views Turck IO-Link masters and all connected IO-Link devices in the network
    • Gets specific information about connected IO-Link devices
    • Executes specific device actions, like Wink, open IO-Link Master in TAS, open IODD Configurator and IO-Link device monitor
    • Filters for specific IO-Link devices
  • RFID Tag Actions HF:
    • Scans UIDs of HF tags and creates a list
    • Gets tag information about tag type, memory size and supported features
    • Reads tag memory in Hex and ASCII
    • Edits data and writes to tag in Hex
    • Sets specific data to the complete or a selected memory area of the tag
  • Diagnostic report to export or print all device settings, status and diagnostics
  • Configuration mode for inductive coupler with IO-link
  • Additional plot for suppression FFT in the Radar Monitor
  • Checks and informs if an addressing conflict occured for the used network adapters
  • Supports factory reset, sets IP address and accesses system settings of TX devices
  • Supports new hardware version of BL20-E-GW-EN and BL67-GW-EN
  • Supports to “go online” with a device automatically, if it isn´t connected to a plc
  • Supports to activate Force Mode to get access to the device, if it is connected to a plc

Changed

  • Improved update process of TAS, which replace the files, stops the old TAS instance and restarts new version of TAS automatically
  • Changes default settings in RFID application “Tag Trace”
  • Devices with running ARGEE or CODESYS programs supports edit device, reset and firmware update
  • Uncheck the import of unique device data by default

Fixed

  • Prevents that process data won´t be updated by using the USB IO-Link Master
  • Size of the navigation bar changed
  • Some text messages and tooltips updated
